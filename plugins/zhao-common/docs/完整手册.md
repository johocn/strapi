# zhao-common 通用工具插件手册

## 概述

zhao-common 是基于 Strapi v5 的通用工具插件，提供配置管理、错误处理、日志记录和国际化支持等基础服务，作为其他插件的基础设施。

**核心功能**:
- 配置管理器（类型安全的配置读取）
- 错误处理器（统一的错误格式和包装）
- 日志服务（标准化日志输出）
- 国际化服务（多语言消息支持）
- 错误码定义（统一的错误码体系）

## 安装与配置

### 1. 安装插件

```bash
npm install zhao-common
```

### 2. 启用插件

在 `config/plugins.ts` 中配置：

```typescript
export default ({ env }) => ({
  "zhao-common": {
    enabled: true,
    resolve: "./node_modules/zhao-common",
  },
});
```

## 服务 API

### Config Manager (`zhao-common.config-manager`)

提供类型安全的配置读取接口。

#### get

获取插件配置项。

**签名**:
```typescript
get<T = unknown>(key: string, defaultValue?: T): T
```

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| key | string | 是 | 配置键名 |
| defaultValue | T | 否 | 默认值 |

**响应**: `T` - 配置值

**示例**:
```typescript
const configManager = strapi.plugin("zhao-common").service("config-manager");
const timeout = configManager.get<number>("timeout", 30000);
```

#### getAll

获取所有配置项。

**签名**:
```typescript
getAll(): Record<string, unknown>
```

**响应**: `Record<string, unknown>` - 所有配置

### Error Handler (`zhao-common.error-handler`)

提供统一的错误处理和格式化。

#### createError

创建 AppError 实例。

**签名**:
```typescript
createError(code: ErrorCode, context?: Record<string, unknown>, message?: string): AppError
```

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| code | ErrorCode | 是 | 错误码 |
| context | Record<string, unknown> | 否 | 上下文参数 |
| message | string | 否 | 自定义消息 |

**响应**: `AppError` - 错误实例

**示例**:
```typescript
const errorHandler = strapi.plugin("zhao-common").service("error-handler");
const error = errorHandler.createError(ErrorCodes.NOT_FOUND, { resource: "user", id: 1 });
```

#### wrapError

包装未知错误为 AppError。

**签名**:
```typescript
wrapError(error: unknown, defaultCode?: ErrorCode): AppError
```

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| error | unknown | 是 | 原始错误 |
| defaultCode | ErrorCode | 否 | 默认错误码（默认 UNKNOWN_ERROR） |

**响应**: `AppError` - 包装后的错误

**示例**:
```typescript
try {
  // 操作
} catch (error) {
  const wrapped = errorHandler.wrapError(error, ErrorCodes.INTERNAL_ERROR);
  throw wrapped;
}
```

#### formatError

格式化错误为 API 响应体。

**签名**:
```typescript
formatError(error: unknown): { code: string; message: string }
```

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| error | unknown | 是 | 错误对象 |

**响应**: `{ code: string; message: string }` - 格式化的错误响应

**示例**:
```typescript
try {
  // 操作
} catch (error) {
  ctx.body = errorHandler.formatError(error);
  ctx.status = 400;
}
```

### Logger (`zhao-common.logger`)

提供标准化的日志输出，自动添加插件前缀 `[zhao-common]`。

#### info

记录信息级别日志。

**签名**:
```typescript
info(message: string, meta?: Record<string, unknown>): void
```

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| message | string | 是 | 日志消息 |
| meta | Record<string, unknown> | 否 | 附加元数据 |

**示例**:
```typescript
const logger = strapi.plugin("zhao-common").service("logger");
logger.info("User logged in", { userId: 123, ip: "192.168.1.1" });
```

#### warn

记录警告级别日志。

**签名**:
```typescript
warn(message: string, meta?: Record<string, unknown>): void
```

**示例**:
```typescript
logger.warn("Low memory warning", { available: "512MB" });
```

#### error

记录错误级别日志。

**签名**:
```typescript
error(message: string, meta?: Record<string, unknown>): void
```

**示例**:
```typescript
logger.error("Database connection failed", { error: err.message });
```

#### debug

记录调试级别日志。

**签名**:
```typescript
debug(message: string, meta?: Record<string, unknown>): void
```

**示例**:
```typescript
logger.debug("Query executed", { query: "SELECT * FROM users", duration: "12ms" });
```

### I18n (`zhao-common.i18n`)

提供国际化消息支持，支持消息模板插值。

#### t

根据错误码获取本地化消息。

**签名**:
```typescript
t(code: string, params?: Record<string, unknown>): string
```

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| code | string | 是 | 消息码 |
| params | Record<string, unknown> | 否 | 模板参数 |

**响应**: `string` - 本地化消息

**内置消息**:

| 消息码 | 消息模板 |
|--------|----------|
| COMMON_001 | 未知错误 |
| COMMON_002 | 参数校验失败: {reason} |
| COMMON_003 | 资源不存在: {resource} |
| COMMON_004 | 无权限访问 |
| COMMON_005 | 认证失败 |
| COMMON_006 | 配置错误: {detail} |
| CHANNEL_001 | 渠道不存在 (id={channelId}) |
| CHANNEL_002 | 渠道层级深度超限（最大 2 级） |
| CHANNEL_003 | 渠道已被禁用 |
| CHANNEL_004 | 邀请码不存在或已过期 |
| CHANNEL_005 | 成员不存在 |
| CHANNEL_006 | 渠道名已存在: {name} |
| CHANNEL_007 | 用户未关联渠道 |
| AUTH_001 | 缺少认证令牌 |
| AUTH_002 | 令牌无效或已过期 |
| AUTH_003 | 角色权限不足 (需要: {roles}) |
| AUTH_004 | 无权访问该渠道 |
| AUTH_005 | 资源所有者不匹配 |

**示例**:
```typescript
const i18n = strapi.plugin("zhao-common").service("i18n");

// 基础使用
const msg = i18n.t("COMMON_001");
// => "未知错误"

// 带参数
const msg = i18n.t("CHANNEL_001", { channelId: 123 });
// => "渠道不存在 (id=123)"

// 带多个参数
const msg = i18n.t("COMMON_002", { reason: "邮箱格式错误" });
// => "参数校验失败: 邮箱格式错误"
```

## 错误类

### AppError

自定义应用异常基类。

**属性**:

| 属性 | 类型 | 说明 |
|------|------|------|
| code | string | 错误码 |
| status | number | HTTP 状态码（默认 400） |
| context | Record<string, unknown> | 上下文参数 |

**方法**:

#### toJSON()

序列化为 API 响应格式。

**示例**:
```typescript
const error = new AppError("NOT_FOUND", { resource: "user" }, 404);
console.log(error.toJSON());
// => { code: "NOT_FOUND", message: "NOT_FOUND", context: { resource: "user" } }
```

### NotFoundError

404 资源不存在异常。

**构造函数**:
```typescript
constructor(resource: string, id?: number | string)
```

**示例**:
```typescript
throw new NotFoundError("user", 123);
// => { code: "USER_NOT_FOUND", status: 404, message: "user not found" }
```

### ForbiddenError

403 无权限异常。

**构造函数**:
```typescript
constructor(code?: string, context?: Record<string, unknown>, message?: string)
```

**示例**:
```typescript
throw new ForbiddenError("FORBIDDEN", {}, "无权访问");
```

### UnauthorizedError

401 认证失败异常。

**构造函数**:
```typescript
constructor(code?: string, context?: Record<string, unknown>, message?: string)
```

**示例**:
```typescript
throw new UnauthorizedError("TOKEN_INVALID", {}, "令牌已过期");
```

### ValidationError

422 参数校验异常。

**构造函数**:
```typescript
constructor(field: string, reason: string, context?: Record<string, unknown>)
```

**示例**:
```typescript
throw new ValidationError("email", "邮箱格式错误");
```

## 使用示例

### 示例1：统一错误处理中间件

```typescript
// middleware/error-handler.ts
export default ({ strapi }) => {
  const errorHandler = strapi.plugin("zhao-common").service("error-handler");
  const logger = strapi.plugin("zhao-common").service("logger");

  return async (ctx, next) => {
    try {
      await next();
    } catch (error) {
      logger.error("Request error", {
        path: ctx.path,
        error: error.message,
        code: error.code,
      });

      const formatted = errorHandler.formatError(error);
      ctx.body = { error: formatted };
      ctx.status = error.status || 500;
    }
  };
};
```

### 示例2：在 Service 中使用错误处理

```typescript
// services/user.ts
export default ({ strapi }) => {
  const errorHandler = strapi.plugin("zhao-common").service("error-handler");
  const i18n = strapi.plugin("zhao-common").service("i18n");

  return {
    async findOne(userId) {
      const user = await strapi.db.query("plugin::users-permissions.user").findOne({
        where: { id: userId },
      });

      if (!user) {
        const error = errorHandler.createError(
          ErrorCodes.NOT_FOUND,
          { resource: "user", id: userId },
          i18n.t("COMMON_003", { resource: "用户" })
        );
        throw error;
      }

      return user;
    },
  };
};
```

### 示例3：日志记录

```typescript
// services/order.ts
export default ({ strapi }) => {
  const logger = strapi.plugin("zhao-common").service("logger");

  return {
    async create(data) {
      logger.info("Creating order", { userId: data.userId, amount: data.amount });

      try {
        const order = await strapi.db.query("order").create({ data });
        logger.info("Order created", { orderId: order.id });
        return order;
      } catch (error) {
        logger.error("Failed to create order", { error: error.message, data });
        throw error;
      }
    },
  };
};
```

## 注意事项

1. **插件依赖**: 其他插件可通过 `strapi.plugin("zhao-common").service()` 获取服务
2. **错误码扩展**: 可通过 `ErrorCodes` 对象扩展自定义错误码
3. **消息扩展**: 可通过 `i18n.setMessages()` 添加自定义消息
4. **日志前缀**: 日志自动添加 `[zhao-common]` 前缀便于区分
5. **错误上下文**: 使用 `context` 参数传递详细的错误信息便于排查
