# zhao-channel 使用手册

## 概述

zhao-channel 是一个基于 Strapi v5 的渠道管理插件，提供**三级渠道层级**（Agent → Regional → Store）、渠道注册、成员管理、权限控制以及分销链体系。

### 核心特性

- **三级渠道层级**：Agent（代理）→ Regional（区域）→ Store（门店）
- **渠道注册**：通过邀请码注册子渠道
- **成员管理**：渠道成员添加/移除/角色变更
- **权限控制**：用户级和角色级的渠道访问授权
- **分销链**：基于邀请码的 3 级分销体系
- **路径追踪**：基于 path 字段的树状结构查询
- **缓存支持**：集成 Redis 缓存加速权限查询
- **异步任务**：基于 Bull 队列的批量授权

## 安装与配置

### 1. 安装插件

```bash
# 在 Strapi 项目根目录下
npm install zhao-channel
```

### 2. 启用插件

在 `config/plugins.ts` 中配置：

```typescript
export default ({ env }) => ({
  "zhao-channel": {
    enabled: true,
    resolve: "./node_modules/zhao-channel", // 本地开发时指向源码路径
  },
});
```

### 3. 依赖要求

- **Strapi**：>= 5.45.0
- **Redis**（可选，用于缓存加速）：需安装 `ioredis`
- **Bull**（可选，用于异步任务）：需安装 `bull`

## 渠道层级结构

渠道采用三级树形结构，每级深度限制如下：

| 层级 | 英语名称 | 层级常数 | 最大深度 |
|------|---------|---------|---------|
| 代理 | Agent | `agent` | 0 |
| 区域 | Regional | `regional` | 1 |
| 门店 | Store | `store` | 2 |

渠道之间通过 `path` 字段记录树路径（如 `/1/3/`），支持高效的子树查询。

## 使用场景

### 场景一：创建渠道树

1. 创建根渠道（Agent 级别）
2. 在 Agent 下创建 Regional 子渠道
3. 在 Regional 下创建 Store 子渠道

### 场景二：渠道注册

新渠道通过上级渠道的邀请码进行注册，系统自动生成新的渠道层级。

### 场景三：成员管理

将用户添加到渠道中，并为用户分配角色（owner / admin / member）。

### 场景四：权限授权

- 给指定用户授权可访问的渠道
- 给指定角色授权可访问的渠道
- 用户可访问的渠道 = 用户直接授权 + 角色授权 + 当前成员渠道

### 场景五：分销体系

- 每个用户拥有唯一邀请码
- 通过邀请码发展下级用户
- 支持 3 级分销链查询
- 每个用户自动关联到其所在渠道

## 管理后台

插件在 Strapi Admin 中提供管理界面，可通过以下菜单访问：

- **Channel**：渠道管理列表
- **Channel Member**：渠道成员管理
- **User Channel**：用户渠道授权
- **Role Channel**：角色渠道授权
- **User Invite**：用户邀请码管理

## 错误码说明

| 错误码 | 说明 |
|--------|------|
| 000201 | 渠道不存在 |
| 030101 | 邀请码不存在或已过期 |
| 030104 | 渠道已被禁用 |
| 030105 | 渠道层级深度超限（最大 7 级） |

---

## 注意事项

1. 渠道深度最大为 2（0 + 1 + 2 共三级），超过将抛出 `030105` 错误
2. 渠道 code 自动生成（8 位大写字母数字组合），也可手动指定
3. 删除渠道会级联删除所有子渠道及相关关联数据
4. 分销链最大深度为 2（共 3 级），超过将另起新的链根
5. Redis 不可用时权限查询降级为数据库查询，不影响核心功能
